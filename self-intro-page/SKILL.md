# self-intro-page — 11の質問で自己紹介ページをつくる

ユーザー（Levela社の社員。非エンジニア前提）に11の質問を1問ずつ聞き、回答をテンプレートに流し込んで `self-intro/index.html` を生成する。プレビュー→対話で修正→Surgeでの公開案内まで伴走する。

## 基本ルール

- **すべて日本語・平易な言葉で。** 専門用語は日常の言葉に言い換える（例:「ターミナル＝文字でパソコンに指示を出す画面」）
- **質問は必ず1問ずつ。** 11個まとめて聞かない。1つ答えてもらってから次へ進む
- **回答が短くてもOK。** 足りなければ「たとえば◯◯みたいなことですか？」と1回だけやさしく深掘りする。無理に聞き出さない
- **勝手に公開しない。** Surgeの公開コマンドは、本人が「公開して」と言ってから実行する
- **個人情報を守る。** 電話番号・住所・社外秘は、本人が書いてきても「公開ページには載せないでおきますね」と外す

## Step 1: あいさつと流れの説明

最初に一言だけ流れを伝える。例:
「これから11の質問を1つずつ聞いていきます。答えるだけで自己紹介ページができあがります。スキップできる質問もあるので、気軽に答えてくださいね」

## Step 2: 11の質問を1問ずつ聞く

以下の順番で、**1問ずつ**聞く。文言はこの通りでなくてよいが、聞く内容は変えない。

| # | 質問 | 備考 |
|---|------|------|
| ① | 名前（ふりがな・ニックネーム可） | ふりがなも一緒に聞く |
| ② | 所属部署・役割 | |
| ③ | 拠点（出身地・現在地） | 例:「沖縄県出身・那覇市在住」 |
| ④ | これまでの略歴 | 年表形式で3〜6行になるように聞く（時期＋できごと） |
| ⑤ | 強み・得意なこと3つ | 3つ揃わなければ「あと◯つ、小さなことでも大丈夫ですよ」と促す |
| ⑥ | 趣味・好きなこと | |
| ⑦ | SNSリンク・発信ジャンル | **任意。**「やっていなければスキップでOKです」と添える |
| ⑧ | 座右の銘・マイルール | |
| ⑨ | 最後にひとこと | ページのしめくくりに載せる、読んだ人へのメッセージ |
| ⑩ | 好きな色・雰囲気 | 色名でも雰囲気の言葉でもOK（例:「青系で爽やか」「落ち着いた感じ」） |
| ⑪ | プロフィール画像 | **任意。**「ChatGPTなどで作った画像や写真を self-intro/ フォルダに入れましたか？」と確認 |

### ⑪プロフィール画像の扱い

- `self-intro` フォルダがまだ無ければ先に作り、「この中に画像ファイルを入れてください」と案内する
- 入れてもらったら `ls self-intro` で中身を確認し、画像ファイル名（png / jpg / jpeg / webp / gif）を控える
- 画像はヒーローのアバターとして `<img src="./ファイル名" alt="（名前）のプロフィール写真">` で埋め込む。**src は index.html と同じフォルダ内のファイル名（`./ファイル名`）のみ許可。外部URLの画像は禁止**
- 画像が無ければ従来どおり、名前の頭文字アバター（CSSのグラデーション円）にする
- **Surgeはフォルダごと公開されるので、フォルダ内の画像も一緒に公開される。** 公開前に「この画像もいっしょに公開されますが大丈夫ですか？」と一言確認する

## Step 3: `self-intro/index.html` を生成する

`templates/template.html` を読み、回答を反映した完成HTMLを **現在の作業フォルダ直下の `self-intro/index.html`** として書き出す（1ファイルで完結・スマホ対応。外部CDN・外部フォントは追加しない。⑪の画像だけは同フォルダ内の相対参照で使ってよい）。

### 差し替え箇所（テンプレ内の `{{...}}` マーカー）

テンプレでは `<!--{{NAME}}-->サンプル値<!--/{{NAME}}-->` のように、コメントで差し替え範囲を示している。マーカーごとサンプル値を回答内容に置き換える（生成後のHTMLにマーカーコメントを残さない）。

| マーカー | 入れる内容 |
|---|---|
| `{{PAGE_TITLE}}` | `<title>` を「（名前）の自己紹介」に |
| `{{AVATAR}}` | ⑪の画像があれば `<img src="./ファイル名" alt="（名前）のプロフィール写真">`、なければ名前の頭文字など1〜2文字 |
| `{{FURIGANA}}` | ①のふりがな（不要なら `<p>` ごと削除） |
| `{{NAME}}` / `{{FOOTER_NAME}}` | ①の名前 |
| `{{ROLE}}` | ②の所属部署・役割 |
| `{{LOCATION}}` | ③の拠点（例:「沖縄県出身・那覇市在住」） |
| `{{TAGLINE}}` | 回答全体から作る本人らしい一言（下記参照） |
| `{{SNS_LINKS_START}}〜END` | ⑦のSNS行。`<a>` を回答のリンク数に合わせて増減。**スキップされたらSTART〜ENDごと削除** |
| `{{GENRE}}` | ⑦の発信ジャンルの一言（SNS行の中。ジャンルの回答がなければ `<p>` ごと削除） |
| `{{TIMELINE_START}}〜END` | ④の略歴を時系列の`<li>`に（3〜6個。時期・見出し・ひとこと） |
| `{{STRENGTH_CARDS_START}}〜END` | ⑤の強み3つを3枚のカードに（見出し＋説明1〜2文） |
| `{{HOBBY_TAGS_START}}〜END` | ⑥の趣味をタグに（1つ1タグで分割） |
| `{{MOTTO}}` / `{{MOTTO_NOTE}}` | ⑧の言葉と補足（補足がなければ`{{MOTTO_NOTE}}`の`<p>`ごと削除） |
| `{{HITOKOTO}}` | ⑨の最後にひとこと |

- **一言（`{{TAGLINE}}`）**: ②〜⑨の回答から、本人の人柄が伝わるキャッチコピーを1文作る。プレビュー時に「一言はこれでいい？」と確認する
- **SNSリンク**: リンク先は本人が答えたURLをそのまま使う。表示名は「サービス名 @アカウント名」が目安
- **文章は盛りすぎない。** 回答にない実績・数字を創作しない。言い回しを整える程度にとどめる

### ⑩の色・雰囲気 → CSSカスタムプロパティの調整

テンプレ冒頭 `:root` の変数（`{{COLOR_THEME}}` の目印つき）だけを変える。CSS本体はさわらない。

| ⑩の答えの例 | --accent | --accent-dark | --accent-soft | --bg |
|---|---|---|---|---|
| オレンジ・元気（初期値） | #ff8a5c | #d95f2f | #fff1e9 | #fffcf8 |
| 青・爽やか | #4a90e2 | #2f6cb8 | #eaf3fd | #f8fbff |
| 緑・ナチュラル | #5cb87a | #3d8f59 | #eaf7ef | #f9fdf9 |
| ピンク・かわいい | #f28bb2 | #d15687 | #fdeef4 | #fffafc |
| 紫・落ち着き | #9d7fd8 | #7357ab | #f3eefb | #fcfaff |

上にない色は、この表と同じ考え方（メイン色／濃いめ／ごく薄い背景／ほんのり色づく地）で調和する4色を選ぶ。文字が読みにくくなる薄すぎる色・まぶしすぎる色は避ける。

## Step 4: プレビューと修正

1. 生成できたら `open self-intro/index.html` でブラウザに表示する（開けない環境なら、ファイルをダブルクリックして開くよう案内する）
2. 「気になるところがあれば『ここを◯◯にして』と教えてください」と伝える
3. 修正要望は対話で1つずつ反映し、そのたびにブラウザで再確認してもらう

## Step 5: Surgeで公開する（本人の合意後）

「公開したい」「Surgeで公開して」と言われたら実行する。先に以下を伝える:

- Surge（surge.sh）は、HTMLページを**無料**でインターネットに公開できるサービス
- 公開すると `https://好きな名前.surge.sh` というURLになり、**誰でも見られる**
- **`self-intro` フォルダごと公開される。** 中に入れたプロフィール画像も一緒に公開される
- 初回だけメールアドレスとパスワードの登録が必要（無料・各自のアカウント）

公開コマンド（`self-intro` フォルダがある場所で実行）:

```bash
npx surge self-intro 好きな名前.surge.sh
```

例: `npx surge self-intro mataken-intro.surge.sh`

- 「好きな名前」の部分は本人に決めてもらう（半角英数字とハイフン。例: `名前-intro`）
- 初回はメールアドレスとパスワードの入力を求められるので、本人に入力してもらう（AIが代わりに決めない）
- 公開が終わったら `https://好きな名前.surge.sh` をブラウザで開いて、本人の目で確認してもらう

## 注意事項（必ず守る）

- **公開＝世界中から見える。** 電話番号・住所・社外秘は載せない。生成時と公開前の2回チェックし、見つけたら外して本人に伝える
- **画像もフォルダごと公開される。** 公開前に、フォルダ内の画像を公開してよいか本人に確認する
- 公開コマンドの実行は本人の合意後のみ（送信ボタンは人が押す）
- テンプレ以外の外部リソース（CDN・Webフォント・外部画像URL）を足さない。1ファイル完結を保つ（例外は⑪のプロフィール画像のみ。同フォルダ内の相対参照 `./ファイル名` に限る）
