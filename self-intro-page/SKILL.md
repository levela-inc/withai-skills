# self-intro-page — 11の質問で自己紹介ページをつくる

ユーザー（Levela社の社員。非エンジニア前提）に11の質問を1問ずつ聞き、回答を `templates/template.html` に流し込んで **`self-intro/index.html`** を生成する。プレビュー→対話で修正→Surgeでの公開案内まで伴走する。

---

## 🚫 最優先ルール（絶対に守る。過去に事故った箇所）

1. **自分で画像を生成しない。** プロフィール画像は「本人が用意した写真」か「頭文字アバター」だけ。
   - **画像生成ツール（Codex / assign-codex-imagegen / DALL·E 等）を絶対に起動しない。** このスキルは「ページを作る」スキルであって「画像を作る」スキルではない。
   - 写真をイラスト化したい人には「ChatGPTなどでご自分で作って、その画像ファイルを self-intro フォルダに入れてください」と案内するだけ（作るのは本人。あなたは待つ）。
   - 画像が無ければ**頭文字アバターで完成させる**。画像は無くても全く問題ない。

2. **必ずページを最後まで作る。** 画像のあるなしに関わらず、11問終わったら**必ず `self-intro/index.html` を生成する**。
   - 画像の受け渡しでつまずいても、そこで止まらない。**先に頭文字アバターでページを作り**、「画像は後から差し替えできます」と伝える。画像待ちでページ作成を止めるのは禁止。

3. **対話は自然な日本語の文章だけ。** `green` `user` `court` などの内部の記号・タグ・ロール名・変数名・色コードを画面に一切出さない。ツールの生の呼び出し文を表示しない。ユーザーに見せるのは、質問と確認の普通の日本語だけ。

4. **修正は1箇所ずつ・最小差分で。** 「ここを直して」と言われたら、**その箇所だけ**を直す。ページ全体を作り直さない（トークン設計なので1箇所直しても他は崩れない）。修正のたびにブラウザで確認してもらう。

（補足ルール）
- **質問は必ず1問ずつ。** 11個まとめて聞かない。1つ答えてもらってから次へ。
- **回答が短くてもOK。** 足りなければ「たとえば◯◯みたいなこと？」と1回だけやさしく深掘り。無理に聞き出さない。
- **勝手に公開しない。** Surgeの公開は本人が「公開して」と言ってから。
- **個人情報を守る。** 電話番号・住所番地・社外秘は、本人が書いてきても公開ページには載せない。

---

## 作業フォルダと出力先（迷子を防ぐ）

- ページは **いま開いている作業フォルダの直下に `self-intro/` フォルダを作り**、その中に `index.html` として書き出す（`self-intro/index.html`）。
- 生成したら、**保存した場所を絶対パスで言葉で伝える**。例:「`~/Desktop/ai-practice/self-intro/index.html` に作りました」。どこにできたか分からない、が一番の事故なので必ず言う。
- プロフィール画像も**同じ `self-intro/` フォルダ**に入れる（Surgeはこのフォルダごと公開するため）。
- ⚠️ **チャットに貼り付けた画像はファイルとして取り出せないことがある。** 画像を使うときは「画像ファイルを Finder で `self-intro` フォルダにドラッグ＆ドロップしてください」と、**ファイルそのものを置いてもらう**よう案内する（チャット貼り付けだけでは進めない）。

---

## Step 1: あいさつ

最初に一言だけ流れを伝える。例:
「これから11の質問を1つずつ聞いていきます。答えるだけで自己紹介ページができあがります。スキップできる質問もあるので、気軽に答えてくださいね」

## Step 2: 11の質問を1問ずつ聞く

| # | 質問 | 備考 |
|---|------|------|
| ① | 名前（ふりがな・ニックネーム可） | ふりがなも一緒に聞く |
| ② | 所属部署・役割 | ページ上部の赤枠チップになる |
| ③ | 拠点（出身地・現在地・勤務地） | グレー枠チップになる。例:「沖縄県出身・那覇市在住」 |
| ④ | これまでの略歴 | 年表形式で3〜6行（時期＋できごと） |
| ⑤ | 強み・得意なこと3つ | 3つ揃わなければ「小さなことでも大丈夫」と促す |
| ⑥ | 趣味・好きなこと | 1つ1タグになる |
| ⑦ | SNSリンク・発信ジャンル | **任意。**「やっていなければスキップでOK」と添える |
| ⑧ | 座右の銘・マイルール | |
| ⑨ | 最後にひとこと | 読んだ人へのメッセージ |
| ⑩ | 好きな色・雰囲気 | 色名でも雰囲気でもOK（例:「青系で爽やか」） |
| ⑪ | プロフィール画像 | **任意。**「写真を self-intro フォルダに入れましたか？無ければ頭文字にします」と確認 |

### ⑦SNSの扱い（任意）
- 回答があれば、ページ内にSNSリンク行を足してよい。無ければ触れない（詰めない）。

### ⑪プロフィール画像の扱い（最優先ルール1・2を厳守）
- 「写真を使いますか？」と聞く。使う場合は「その画像ファイルを `self-intro` フォルダに入れて『入れた』と教えてください」と案内。
- 入れてもらったら `ls self-intro` で中身を確認し、画像ファイル名（png/jpg/jpeg/webp/gif）を控える。
- 画像はアバターとして `<img src="./ファイル名" alt="（名前）のプロフィール写真">` で埋め込む（**src は同フォルダ内のファイル名だけ。外部URL禁止**）。
- **画像が無い／用意が面倒／後でいい → 頭文字アバターで即ページを作る。** 「あとで画像ファイルを self-intro フォルダに入れて『入れた』と言ってもらえれば差し替えます」と添える。
- **写真を絵にしたい人へ**: 「ChatGPTなどで作った画像ファイルを self-intro フォルダに入れてください。こちらでは画像は作りません」とだけ案内する。

## Step 3: `self-intro/index.html` を生成する

`templates/template.html` を読み、回答を反映した完成HTMLを `self-intro/index.html` として書き出す（1ファイル完結・スマホ対応。外部CDN・外部フォントは足さない。例外は⑪の画像＝同フォルダ内の相対参照のみ）。

### 差し替えマーカー（`<!--{{NAME}}-->サンプル<!--/{{NAME}}-->` のサンプルを回答で置き換え、マーカーコメントは残さない）

| マーカー | 入れる内容 |
|---|---|
| `{{PAGE_TITLE}}` | `<title>` を「（名前）の自己紹介」に |
| `{{AVATAR}}` | 画像ありは `<img src="./ファイル名" ...>`、無しは名前の頭文字1〜2文字 |
| `{{NAME}}` / `{{FOOTER_NAME}}` | ①の名前 |
| `{{FURIGANA}}` | ①のふりがな（不要なら`<p>`ごと削除） |
| `{{ROLE_CHIPS}}` | ②の所属・役割を赤枠チップ（`<span class="chip role">`）に。「／」「・」で分けて増減 |
| `{{FACT_CHIPS}}` | ③の拠点などをグレー枠チップ（`<span class="chip fact">`）に。無ければ削除 |
| `{{TAGLINE}}` | 回答全体から本人らしい一言を1文（プレビュー時に確認） |
| `{{TIMELINE}}` | ④の略歴を`<li>`に（3〜6個。時期・見出し・ひとこと） |
| `{{STRENGTH_CARDS}}` | ⑤の強み3つを3枚のカードに（見出し＋説明1〜2文） |
| `{{LIKE_TAGS}}` | ⑥の趣味を1つ1タグに。**アイコンも選ぶ（下記）** |
| `{{MOTTO}}` / `{{MOTTO_NOTE}}` | ⑧の言葉と補足（補足なければ`{{MOTTO_NOTE}}`の`<p>`ごと削除） |
| `{{HITOKOTO}}` | ⑨の最後にひとこと |

### ⑥趣味アイコンの選び方（すきなことタグ）
テンプレ上部にアイコン一覧（`<symbol id="ico-xxx">`）がある。各タグの `<use href="#ico-xxx">` の **xxx を趣味に合う名前に変える**だけ。
- music(音楽/歌/カラオケ) camera(写真) book(読書/勉強) coffee(カフェ/お茶) travel(旅行/おでかけ) fitness(筋トレ/ラン/ジム) game(ゲーム) film(映画/ドラマ/アニメ) food(料理/グルメ/食べ歩き) pet(犬/猫/動物) sport(スポーツ全般) art(絵/デザイン/ものづくり) nature(アウトドア/キャンプ/登山/釣り) heart(推し/好きなもの)
- **どれにも合わなければ `star` を使う**（必ず何か付く＝崩れない）。

### ⑩色テーマの調整
テンプレ冒頭 `:root` の色変数だけを、⑩の答えに合わせて変える（CSS本体はさわらない）。テンプレ内コメントの早見表（レッド初期値／ブルー／グリーン／オレンジ／パープル／ピンク）からコピー。表に無い色は「メイン色／濃いめ／ごく薄い地／淡い罫線」の4色を調和させて作る。

- **一言（{{TAGLINE}}）**: 回答全体から人柄が伝わる一文を作り、プレビュー時に「この一言でいい？」と確認。
- **回答にない実績・数字を創作しない。** 言い回しを整える程度にとどめる。

## Step 4: プレビューと修正

1. `open self-intro/index.html` でブラウザに表示（開けない環境ならダブルクリックで開くよう案内）。
2. **保存場所を絶対パスで伝える**（「作業フォルダと出力先」参照）。
3. 「気になるところは『ここを◯◯にして』と教えてください」と伝える。
4. 修正要望は**1箇所ずつ最小差分**で反映し、そのたびにブラウザで再確認してもらう（最優先ルール4）。

## Step 5: Surgeで公開する（本人の合意後）

「公開して」と言われたら実行する。先に伝える:
- Surge（surge.sh）は、HTMLページを**無料**で公開できるサービス。公開すると `https://好きな名前.surge.sh` になり、**誰でも見られる**。
- **`self-intro` フォルダごと公開される**（中の画像も一緒に公開）。
- 初回だけメールアドレスとパスワードの登録が必要（無料）。

公開コマンド（`self-intro` フォルダがある場所で実行）:
```bash
npx surge self-intro 好きな名前.surge.sh
```
例: `npx surge self-intro mataken-intro.surge.sh`
- 「好きな名前」は本人が決める（半角英数字とハイフン）。
- 初回のメール／パスワード入力は**本人にしてもらう**（AIが代わりに決めない）。
- `npx` が使えない（Nodeが未インストール）場合は、その旨を伝えて Node.js のインストールから案内する。
- 公開後 `https://好きな名前.surge.sh` を開いて、本人の目で確認してもらう。

## 注意事項（必ず守る）

- **公開＝世界中から見える。** 電話番号・住所番地・社外秘は載せない。生成時と公開前の2回チェックし、見つけたら外して本人に伝える。
- **写真に第三者（家族・子ども等）が写っている場合**は、公開ページに載せてよいか一言確認する（本人だけに切り抜く選択肢も伝える）。
- テンプレ以外の外部リソース（CDN・Webフォント・外部画像URL）を足さない。1ファイル完結を保つ（例外は⑪の画像＝同フォルダ内の相対参照のみ）。
- アイコン置き場の `<svg>...<symbol></svg>` ブロックは消さない（消すとアイコンが出なくなる）。
